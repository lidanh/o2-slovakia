# Parallel Category-Based Evaluation System

## Overview

Replace the current single-LLM-call, 5-dimension feedback system with a parallel, 6-category evaluation system that matches O2 Slovakia's official Evaluation Criteria XLSX checklist. Each category gets its own focused GPT-4o prompt, all run in parallel, with deterministic score aggregation.

## Evaluation Categories (from XLSX)

| # | Category | Key | Weight | Items | Scoring |
|---|----------|-----|--------|-------|---------|
| 1 | Standards & Communication Structure | `communication_standards` | 10% | 4 | Items 1,2,4: Pass/Fail. Item 3: Pass/Partial/Fail |
| 2 | Active Listening & Perception | `active_listening` | 15% | 3 | All: Pass/Partial/Fail |
| 3 | Solution | `solution` | 30% | 5 | All: Pass/Partial/Fail |
| 4 | Attitude | `attitude` | 15% | 2 | All: Pass/Partial/Fail |
| 5 | Sales | `sales` | 30% | 7 | All: Pass/Partial/Fail. Frontline scenarios only |
| 6 | WAU Effect | `wau_effect` | bonus | 1 | Bonus 0-20% on top of final score |

### Scoring Model

- Item scores: `passed` = 2pts, `partially_passed` = 1pt, `failed` = 0pts
- Category score = `sum_of_item_scores / max_possible_points` (as 0-100%)
- For leadership (non-sales) scenarios: Sales weight (30%) redistributed proportionally among categories 1-4
- Final score = weighted sum of category percentages (0-100)
- WAU bonus: adds 0-20% on top (capped at 100)
- Star rating: 90-100=5, 75-89=4, 60-74=3, 40-59=2, <40=1

### Scenario Type Mapping

- `frontline` scenarios → Sales evaluation included (30% weight)
- `leadership` scenarios → Sales excluded, weight redistributed

## Architecture

### Parallel LLM Evaluation

```
generateFeedback(transcript, scenario, difficulty, scenarioType)
  → evaluator.evaluateAll(transcript, context, scenarioType)
    → Promise.all([
        evaluateCategory("communication_standards", ...),
        evaluateCategory("active_listening", ...),
        evaluateCategory("solution", ...),
        evaluateCategory("attitude", ...),
        evaluateCategory("sales", ...),       // only for frontline
        evaluateCategory("wau_effect", ...),
      ])
    → scoring.calculateFinalScore(categoryResults, scenarioType)
    → merge highlights & suggestions from all categories
  → return FeedbackResult
```

Each `evaluateCategory()` is an independent GPT-4o call with:
- Full transcript + scenario context
- Focused system prompt for that category only
- Structured JSON response with per-item scores + feedback
- Temperature: 0.3

### Error Handling

If a single category evaluator fails, the system retries once. If it fails again, that category is marked as "evaluation_error" and excluded from the final score calculation (weights redistributed among successful categories).

## Data Model

### New FeedbackBreakdown (JSONB)

```typescript
type ItemScore = "passed" | "partially_passed" | "failed";

interface FeedbackBreakdown {
  categories: {
    [key: string]: {
      weight: number;
      score_percentage: number;
      items: {
        key: string;
        label: string;
        score: ItemScore;
        max_points: number;
        earned_points: number;
        feedback: string;
      }[];
      highlights: SessionHighlight[];
      suggestions: string[];
    };
  };
  wau_bonus_percentage: number;
  sales_included: boolean;
}
```

No database migration needed — `feedback_breakdown` is already JSONB. Old data can be wiped (not production yet).

## UI Design

### Session Detail Page

1. **Score Card** — Same position, score (0-100) + star rating (1-5), both deterministically derived
2. **Feedback Summary** — Generated by merging category summaries
3. **Category Breakdown** — Collapsible accordion:
   - Header: category icon + name + weight badge + percentage score bar
   - Expanded: per-item rows with Pass/Partial/Fail color-coded badges + feedback text
4. **Radar Chart** — Updated to 6 categories (or 5 for non-sales)
5. **WAU Bonus** — Special bonus badge/indicator when earned
6. **Suggestions & Highlights** — Aggregated from all categories, same layout as today

## Prompt Design Principles

Each category prompt:
- Starts with the same IMPORTANT CONTEXT block (training platform, user = customer being evaluated)
- Specifies exactly which sub-metrics to evaluate with their exact assessment questions from the XLSX
- Defines the exact JSON response schema with per-item scoring
- Instructs the LLM to cite specific transcript moments in feedback
- Uses the same temperature (0.3) for consistency
